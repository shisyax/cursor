# Agents.md

このリポジトリはドキュメントプロジェクトです。

## 基本方針

- テストは不要です。
- PR 本文に `Tests` セクションは書きません。
- Markdown を更新するときは、日本語の文体を必ず確認します。

## 日本語文体

- 本文は敬体で統一します。
- `です` と `ます` は、述語に合わせて自然に使い分けます。
  - 名詞で終わる文は、必要に応じて `です` を使います。
  - 動詞で終わる文は、必要に応じて `ます` を使います。
  - 形容詞に機械的に `です` を付けるだけの文は避けます。
  - 例: `美しいです。` のように不自然な場合は、主観を避けて、`造形に美しさがあります。`、`美しい形状をしています。`、`装飾が施されています。` のように文脈に合う客観的な説明へ直します。
  - 名詞を当てはめただけの説明は避けます。文脈上確認できる動きや描写が分かる文に直します。
  - 単純な行動説明だけで終わらせません。その行動が物語や設定の中で持つ役割まで書きます。
  - 抽象名詞に頼った説明は避け、文脈上確認できる特徴や描写を具体的に説明します。
- 状態や描写を説明する受身表現では、`〜れています`、`〜られています`、`〜されています` の形を使います。
- 例示は限定列挙ではありません。`描かれています`、`扱われています`、`使われています`、`置かれています`、`位置づけられています`、`変身させられています` など、文脈に合う形を選びます。
- `描かれます`、`扱われます`、`使われます` のような語尾は、未来の動作、習慣的な動作、一般的な手順を表す場合に使います。状態説明では使いません。
- 曖昧な傾向表現は避け、文脈上確認できる描写に直します。
- 対比表現を続けると説明がくどくなるため、多用しません。肯定文や具体的な描写に置き換えます。
- 箇条書きでも、説明文として書く場合は語尾を省略しません。
- 箇条書きの説明量が偏らないようにします。短い項目が目立つ場合は、文脈、描写、影響を補って他の項目と粒度を揃えます。

## PR 本文

- PR 本文に修正点、レビュー対応、作業履歴は書きません。
- PR 本文の Summary には、最終的に追加または変更された成果物だけを書きます。
- 文体修正、指摘対応、細かい調整の列挙はマージコメントとして不適切なので書きません。

## 最終報告

- 最終報告は、実施した成果だけを簡潔に書きます。
- 「確認のみ行いました」「追記していません」のような報告だけの文は書きません。
- 「テストは実行していません」のような実施有無だけの文は書きません。
- ユーザーが求めていない補足説明は省きます。
